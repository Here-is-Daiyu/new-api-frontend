# New API 现代化轻量前端

一个面向 **New API** 的轻量管理界面，适合日常查看与管理账号、Token、模型和日志信息。

> 说明：本项目绝大部分代码由 AI 编写。

## 功能特性

- **自定义 BaseURL**：支持填写并保存目标服务地址。
- **账号登录**：支持邮箱 / 用户名 + 密码登录。
- **用户信息查看**：登录后可查看当前账号状态、分组与额度信息。
- **控制台日志快捷跳转**：登录后可从顶部用户栏一键在新标签页打开目标 New API 的 `/console/log` 页面。
- **Token 管理**：支持查看、搜索、创建、编辑、启用 / 禁用与删除。
- **Token Key 默认隐藏**：默认不展示任何前缀或片段，点击查看时才显示完整 Key；复制时仍使用完整值。
- **可用模型查看**：支持查看账号全量可用模型，或按指定 Token Key 精确查看对应可用模型，并在 Key 模式下展示所属方信息。
- **日志审计**：支持查看调用日志、基础筛选与统计信息；后台每 2 秒自动加载更多日志，刷新时保持旧数据显示直到新数据就绪；统计卡片下方展示彩色模型消耗比例横条，横条数据改由 `/api/data/self` 提供并按 `model_name` 聚合 `quota`，鼠标悬停可查看具体占比。
- **主题切换**：支持浅色 / 深色主题切换。
- **移动端适配**：在手机端提供更紧凑的卡片式布局，并修复真实手机浏览器下整页滚动与错误缩放为桌面宽度的回归问题。

## 基础使用方式

1. 打开页面后，先填写并保存目标服务地址。
2. 使用邮箱 / 用户名和密码登录。
3. 在“令牌管理”中查看或维护 Token。
4. 如需进入目标 New API 官方控制台日志页，可点击顶部用户栏中的“控制台日志”。
5. 在“可用模型”中按账号全量或指定 Token Key 查看当前可访问的模型列表。
6. 在“日志审计”中查看调用记录、统计信息与基础筛选结果。

## 安全说明

- Token Key 默认隐藏，降低界面误暴露风险。
- 地址输入仅支持 `http` / `https`。
- 页面会进行基础输入校验，避免明显无效的空请求。

## 使用注意事项

- 可用模型面板默认读取当前登录账号可访问的模型列表，也支持切换到指定 Token Key，精确查看该 Key 可访问的模型并显示 `owned_by` 信息。
- Token Key 默认隐藏；仅在手动点击查看时显示完整值，复制操作始终使用完整值。
- 手机端下 `Token` / `日志审计` 会自动切换为更紧凑的布局，优先保证核心信息可直接查看；页面整体应可直接纵向滑动，不再依赖桌面宽度缩放。
- 刷新 Token、模型或日志时，页面会尽量保留当前内容与浏览上下文，待新结果返回后再更新。
- "控制台日志"按钮会按当前 BaseURL 在新标签页打开目标 `/console/log`，浏览器会自动携带该目标域名下已存在的 Cookie；若目标域名尚无有效登录态，仍可能需要在官方页面重新登录。
- 若手机端部署后仍看到旧的桌面缩放样式，请强制刷新页面，确保新的首页 viewport 配置与静态资源版本参数已生效。
- **日志审计后台加载**：进入日志审计页面后，系统会每 2 秒自动在后台加载新日志，不会打断当前浏览；离开日志页面时自动停止后台加载。
- **日志刷新合并**：点击刷新按钮或筛选时，系统会先从后端拉取完整日志数据，然后再更新页面显示，不会清空旧数据直到新数据就绪。
- **日志滚动加载**：向下滚动日志列表，当剩余未加载日志少于 100 条时，系统会自动加载下一批 200 条日志。
- **模型消耗比例**：统计卡片下方的彩色横条通过 `/api/data/self` 获取数据；默认统计近 24 小时，若日志筛选中填写了开始/结束时间，则优先按该时间范围请求，并对返回结果按 `model_name` 聚合 `quota` 后展示各模型消耗占比；鼠标悬停在某个颜色段上可查看该模型的具体占比和消耗额度。
